Our Conclusions

How did Flask and Semantic UI perform?

Semantic UI was quite easy to pick up and use. It was similar to bootstrap, so our previous experince with that framework was useful.

How was installation and configuration?

Installation was notably easier on computers using linux than those using windows. Linux only required a few commands to get Flask and Semantic UI working, while the process for windows involved external installers, and literal hours of research and fiddling in order to just get the most basic elements of the frameworks working.

How easy was is to create a page?

Creating a page using Semantic UI was very easy. However, the structure was similar to bootstrap, as mentioned above, which might make how easy it was a bit biased. Flask was a bit harder to get a hang of using within a webpage than some other server-side frameworks, such as PHP.

How was the learning curve?

Semantic Ui was easy to pick up, which was quite surprising. When researching what types of developers use Semantic UI, many people mentioned that it was harder to pick up than most other front-end frameworks. We didn't find that to be the case. Flask was often mentioned as being easier to learn than Django, another python based framework, but we found it to be harder to grasp than most other frameworks we explored within this course.

How would we have done things differently?

Using more computers with linux as the operating system would have made the installation of our frameworks much quicker and easier. Starting off with frustrations about such a small, basic thing definitely brought a bit of pessimism into the project.

Regrets on the frameworks of choice?

Semantic UI worked well, and it was quick to pick up. Perhaps a different server-side framework that was more familiar would have been a better choice. We wanted to try something new to expand our skills in web development, but perhaps this wasn't the place to jump directly into the deep end.
