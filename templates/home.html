Overview: Semantic UI and Flask

Semantic UI

Semantic is a development framework that helps create beautiful, responsive layouts using human-friendly HTML

Popularity

Semantic UI is not as popular as Bootstrap, and has a larger learning curve.

Strengths

Load only the components you need

Semantic UI package is well organized, with every component stored into it's own stylesheet and JavaScript file. This makes it easy to load only certain packages which decreases both file size and access time

The elements offer a huge amount of customization compared to other frameworks such as Bootstrap
Its semantic

It uses a semantic naming scheme for the styling, which makes it easy to understand for beginners

Beautifully designed

Semantic UI has a futuristic and beautiful design that many people will find appealing.

It has many UI components

It has many UI components that can suit many different types of projects.

Good documentation

The documentation is well written and very easy to use due to the many examples provided in the source code.

Weaknesses

Very large file size

The packages provided by Semantic UI are much larger in comparison to Boostrap. Using specific components is recommended rather than using all the components.

Buggy

There are many UI intricacies such as incorrect position and unable to mix classes.

Flask

Flask is a micro framework. The aim of the framework is to keep the core simple but extensible. Flask is easy to learn quickly, and is also easy to tweak to the developers needs.

Popularity

Flask is very popular currently, even more so than Django which is another python framework.

Strengths

Extremely flexible
Minimalist without sacrificing power
Simple to learn and use
Routing URLs is easy
The Core of the framework is small and easily extensible.

Weaknesses

Not Async friendly
Limited support and documentation
Lack of database and forms
Truely limited in features

Types of Developers

Flask is often used in smaller projects that have opted to use python. Flask is also useful for small projects due to being lightweight and easy to deploy
